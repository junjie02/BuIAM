# Reports Directory

`reports/` 用来存放项目开发过程中的阶段性记录，主要是日报、阶段总结和里程碑说明。

## 命名规则

- 日报：`YYYY-MM-DD.md`
- 阶段总结：`phase-{name}.md`
- 里程碑记录：`milestone-{name}.md`

示例：

- `2026-04-26.md`
- `phase-real-agent-integration.md`
- `milestone-feishu-openapi-poc.md`

## 建议内容结构

建议每份报告至少包含这些部分：

1. 今日目标或本阶段目标
2. 完成内容
3. 修改文件
4. 测试与验证结果
5. 当前问题
6. 后续计划

## 编写原则

- 记录“做了什么”和“为什么这么做”，不要只写结论。
- 尽量附上关键文件路径，方便后续追溯。
- 测试结果要尽量具体，例如写明执行了哪条命令。
- 如果存在阻塞项，要明确说明是代码问题、环境问题还是配置问题。
